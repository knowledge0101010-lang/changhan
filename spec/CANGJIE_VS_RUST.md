# 창제 벤치마크 ≠ Rust / Han

창한 한글화는 **Han UX(표면 완전한글)** 를 따르되, **타깃 문법은 창제**다.  
Rust/Han 문법을 그대로 옮기면 `cjc`가 거부한다.

## 표면 UX (Han과 동일 목표)

| 층 | Han | 창한(목표) |
|----|-----|------------|
| 키워드·타입·리터럴 | 한글 | 한글 → 창제 영문 키워드로 번역 |
| 내장 (`출력` 등) | 한글 런타임 | 한글 → `println` 등 |
| 식별자 | 한글 가능 | 통과 후 창제 허용 범위 VERIFY |
| 논리 | `그리고`/`또는` | → `&&`/`\|\|` |
| 어순 | `만약 조건 이면` | → `if (조건)` (이면 제거) |
| 산술·비교·대입 | C 기호 | C 기호 유지 |
| 컴파일러 본체 | Rust | **번역 엔진** + 공식 `cjc` |

## 창제 ≠ Rust (반드시 다름)

| 항목 | Rust / Han | 창제 (번역 결과) |
|------|------------|------------------|
| 함수 | `fn` / Han `함수`→자체 IR | `func` · 반환타입은 `: T` (화살표 `->` 아님) |
| 가변/불변 | `let mut` / `let` | `var` / `let` |
| 조건 괄호 | 선택(Rust) / Han 자유 | **필수** `if (cond)` |
| 상속/구현 | `:` / `impl` | `<:` · `extend` |
| 동시성 | async/thread | `spawn { => ... }` |
| Option | `Option` | `Option` + 자동 Some 래핑 |
| 패키지 | `mod`/`use` | `package` / `import` |
| main | `fn main()` | `main()` (패키지 엔트리) |
| 섀도잉 | 같은 스코프 가능(Rust) | **같은 스코프 재선언 금지** |
| struct/class | struct≈소유권 | struct=값복사, class=참조공유 |
| match | `match x { }` | `match (x) { case … => }` |
| 파이프 | Han `\|>` | 창제도 `\|>` / `\~>` 있음 — 통과 |

## 번역 엔진이 추가로 하는 창제 보정

1. `만약 COND 이면 {` → `if (COND) {` (`이면` 삭제, 괄호 삽입)
2. `함수 name(args) -> T` → `func name(args): T` (또는 main이면 `main()`)
3. `동안 COND` / `COND 동안` → `while (COND)`
4. `반복 x 안에서 xs` → `for (x in xs)`
5. `그리고`/`또는` → `&&`/`||`
6. Han `구조`/`구현` → 창제 `struct` / (메서드는 타입 내부 `func` 또는 `extend`)
7. `포함 "a.hgl"` → `import …` (모듈 경로 창제 규칙)

## 결론

- **한글화 벤치마크 대상 = 창제 프로그래밍 언어**
- **UX 벤치마크 = Han 표면 완전한글**
- **Rust와 다른 점 = 위 표** — 창한은 Rust 한글판이 아니라 **창제 한글 표면**
