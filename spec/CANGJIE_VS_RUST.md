# 창제 벤치마크 ≠ 러스트 / 한(Han)

창한의 한글화는 **한(Han)의 표면 UX(완전 한글)** 를 따르되, **목표 문법은 창제**입니다.  
러스트·한 문법을 그대로 옮기면 `cjc`가 거부합니다.

## 표면 UX (한과 같은 목표)

| 층 | 한(Han) | 창한(목표) |
|----|---------|------------|
| 키워드·타입·리터럴 | 한글 | 한글 → 창제 영문 키워드로 번역 |
| 내장 (`출력` 등) | 한글 런타임 | 한글 → `println` 등 |
| 식별자 | 한글 가능 | 통과 후 창제 허용 범위에서 검증 |
| 논리 | `그리고` / `또는` | → `&&` / `\|\|` |
| 어순 | `만약 조건 이면` | → `if (조건)` (`이면` 제거) |
| 산술·비교·대입 | C 기호 | C 기호 유지 |
| 컴파일러 본체 | 러스트 | **번역 엔진** + 공식 `cjc` |

## 창제 ≠ 러스트 (반드시 다름)

| 항목 | 러스트 / 한 | 창제 (번역 결과) |
|------|-------------|------------------|
| 함수 | `fn` / 한 `함수`→자체 IR | `func` · 반환 타입은 `: T` (`->` 아님) |
| 가변 / 불변 | `let mut` / `let` | `var` / `let` |
| 조건 괄호 | 선택(러스트) / 한은 자유 | **필수** `if (조건)` |
| 상속·구현 | `:` / `impl` | `<:` · `extend` |
| 동시성 | async / thread | `spawn { => ... }` |
| Option | `Option` | `Option` + 일부 자동 Some 래핑 |
| 패키지 | `mod` / `use` | `package` / `import` |
| 진입점 | `fn main()` | `main()` (패키지 엔트리) |
| 그림자 선언 | 같은 범위에서 가능(러스트) | **같은 범위 재선언 금지** |
| struct / class | struct≈소유권 | struct=값 복사, class=참조 공유 |
| match | `match x { }` | `match (x) { case … => }` |
| 파이프 | 한 `\|>` | 창제도 `\|>` / `\~>` — 그대로 통과 |

## 번역 엔진이 하는 창제 보정

1. `만약 조건 이면 {` → `if (조건) {` (`이면` 삭제, 괄호 삽입)
2. `함수 이름(인자) -> 타입` → `func 이름(인자): 타입` (본문은 `main()`으로 정규화)
3. `동안 조건` / `조건 동안` → `while (조건)`
4. `반복 x 안에서 목록` → `for (x in 목록)`
5. `그리고` / `또는` → `&&` / `||`
6. 한 `구조` / `구현` → 창제 `struct` / (메서드는 타입 안 `func` 또는 `extend`)
7. `포함 "파일.hgl"` → `import …` (창제 패키지 경로 규칙)

## 결론

- **한글화의 문법 기준 = 창제**
- **UX 기준 = 한의 표면 완전 한글**
- **러스트와 다름 = 위 표** — 창한은 러스트 한글판이 아니라 **창제 한글 표면**
